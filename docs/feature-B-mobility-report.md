# Feature B — Dedicated Stretch & Mobility AI Report

> Goal: give mobility/stretch the same dedicated AI Coach report the Workout page gets — a real
> feedback loop, not a side note — with recommendations that update the routine via one-tap
> approve/dismiss cards.

As-built spec. The three design questions were settled in an interview (2026‑06‑14): see §1.

## 1. Decisions
1. **Lives in the Coach tab.** A **Workout / Mobility** report-type switch on the Coach home screen.
   Reuses the existing key/model/range/length controls, API plumbing, and report history.
2. **Add / lengthen / drop.** The report can recommend adding a drill, lengthening a hold, and
   dropping a neglected/redundant drill — all as approve-or-dismiss cards.
3. **Shared, tagged history.** Reports save into the same `coach_reports` store tagged
   `kind:'workout'|'mobility'`, with an All / Workout / Mobility filter and a per-row badge.

## 2. The report
Generated by its own system prompt + payload; markdown sections, in order:
**Consistency** (mobility streak + days hit in range) · **Coverage & Balance** (per-area drill/seconds
coverage + flagged tight areas; calls out zero-coverage areas) · **Hold Durations** · **Match to
Training** (the weekly lifting split → what mobility the training demands) · **Recommendations**.

`coachMobilityPayload()` sends: `mobilityStreak`, `mobilityDaysHit`/`daysInRange`, `tightAreas`
(reused from Feature A), `mobilityRoutine` + `stretchRoutine` (names/seconds/areas), `areaCoverage`
(per canonical area), `availableMobility`/`availableStretches` (catalog drills to draw adds from), and
`trainingFocus` (the five day-template tags).

## 3. Approvable mobility edits
The model appends a strict ```json `{"edits":[…]}` block (`COACH_MOB_EDIT_INSTR`). Each edit is
`{type:"mobility"|"stretch", action:"add"|"remove"|"duration", drill, seconds, why}`.

- `coachMobValid()` validates every edit against the catalog/active list: exact name match, drill must
  exist (no inventions), adds must come from the available list, removes/durations must target an
  active drill, durations bounded 30–300s and must differ, never remove the last drill.
- `coachMobParseEdits()` strips the JSON from the prose and de-dupes.
- Edits carry an **`e.mob` flag** and flow through the **same** approve/dismiss/undo cards as the
  workout edits — `coachDoApprove`/`coachUndoEdit`/`coachEditsHTML`/`coachEditBanner`/`coachEditKey`
  branch on `e.mob`. `coachMobApprove`/`coachMobUndo` apply to the active lists:
  - **add** → `saveActiveList` (inserted next to related drills via `stSuggInsertPos`, from Feature A)
  - **remove** → splice from the active list (Undo restores at its index)
  - **duration** → `setStretchTime` global hold override (Undo restores the prior value)

`stretch` edits target the **`floor`** routine (the daily 10‑min stretch list); `mobility` edits
target the mobility list.

## 4. Touch points (all in `index.html`, plus `sw.js` cache bump + `test/smoke.js`)
- **State:** `coachKind`, `coachHistFilter`; `coachSetKind`, `coachSetHistFilter`.
- **Analysis:** `coachMobilityPayload`, `coachMobilitySystem`, `coachMobilityPrompt`,
  `coachMobLastReport`, `coachAnalyzeMobility`; `coachAnalyze` branches on `coachKind`.
- **Edits:** `COACH_MOB_EDIT_INSTR`, `coachMobValid`, `coachMobParseEdits`, `coachMobApprove`,
  `coachMobUndo`, `coachMobModeLabel`; `coachGetParsed` dispatches by report kind; the shared edit
  functions branch on `e.mob`.
- **Render:** home report-kind switch (+ Focus hidden for mobility, button relabel); history filter +
  `cch-htag` badges; workout reports now tagged `kind:'workout'`.

## 5. Tests (in `test/smoke.js`)
Payload shape (routine/coverage/available/training/consistency); system prompt + edit instruction;
edit validation (add/duration/remove + rejection of invented drills, out-of-range durations, and
last-drill removal); parse + `coachGetParsed` kind dispatch; apply + undo for add and duration; home
render hides Focus and relabels the button for the mobility report.
